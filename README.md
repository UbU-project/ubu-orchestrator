# ubu-orchestrator

`ubu-orchestrator` is the local async Rust service layer for the UbU Phase 1
dogfooding loop. It connects the store, GitHub adapter, planning kernel, and UI
through a loopback HTTP API.

The HTTP server binds to `127.0.0.1` only. This is intentional for Phase 1.

## Non-goals

- No UI rendering.
- No cloud backend.
- No multi-device sync.
- No autonomous GitHub writes.
- No broad personal data ingestion.

## GitHub tokens

Developer mode reads `GITHUB_TOKEN`. Desktop session mode accepts a pasted token
from the UI for the current process. Tokens are kept in memory only, never
persisted, and never logged.

## Development

```sh
cargo fmt --check
cargo clippy --all-targets
cargo test
```

Local sibling crate overrides may be generated by the devshell in
`.cargo/config.toml`. That file is intentionally gitignored; committed
dependencies use Git revisions only.
